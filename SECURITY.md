# Security Policy

## Supported Versions

CodeGuard is actively maintained and security updates are provided for the latest stable releases.

| Version | Supported |
| ------- | --------- |
| Latest release | ✅ Supported |
| Previous minor release | ✅ Security fixes only |
| Older versions | ❌ Not actively supported |

We recommend always using the latest available version to receive security improvements, bug fixes, and new security rules.

---

## Reporting a Vulnerability

Security is a core focus of CodeGuard. If you discover a vulnerability, security issue, or potential weakness, please report it responsibly.

Please do not create a public GitHub issue for security vulnerabilities.

### How to Report

Send a detailed report through:

- GitHub Security Advisories
- Email: yogeshd278@gmail.com

Include the following details:

- Description of the vulnerability
- Steps to reproduce the issue
- Affected version
- Example code or configuration (if applicable)
- Potential impact
- Suggested mitigation (if available)

---

## Response Process

After receiving a vulnerability report:

1. We will acknowledge the report within **7 days**.
2. The issue will be reviewed and validated.
3. We will investigate the impact and affected components.
4. A fix or mitigation plan will be prepared.
5. Security updates will be released when appropriate.

---

## Responsible Disclosure

We request that security researchers:

- Do not publicly disclose vulnerabilities before a fix is available.
- Avoid accessing or modifying other users' data.
- Avoid actions that may impact availability of services.
- Provide sufficient information to reproduce the issue.

We appreciate responsible security research and will credit contributors who help improve CodeGuard security.

---

## Security Scope

CodeGuard security reports may include issues related to:

- Static code analysis engine
- Security rule detection
- Dependency scanning
- CI/CD integrations
- Report generation
- Authentication and authorization (if applicable)
- Data handling and privacy

---

## Out of Scope

The following are generally not considered security vulnerabilities:

- False positives in code analysis rules
- Feature requests
- Performance improvements without security impact
- Issues requiring unrealistic user actions

---

## Security Updates

Security fixes will be documented through:

- GitHub Security Advisories
- Release notes
- Changelog updates

Thank you for helping make CodeGuard safer for developers and teams.
